O.O.P. | Pseudocode | MVC

Pseudocode
Pseudocode is my process of writing code in an "english" form. Code in an easily readable form. Pseudocode should be used to define the functionality, objects, functions, classes, and variables that will be needed to construct my webpage.
Pseudocode in OOP is helpful for me to work from "the outside, in". Setting Classes then moving "in" to variables, functions, helper functions, objects, and properties is a great structure form to work through my functionality, and establish the process of my webpage.
Singleton design pattern means there is only one instance of a class. Often, State Objects are a Singleton instead of glabal to cut down of confusion when many global items will be used. This is abetter structured way of programming when it is applicable. By declaring the constructors of a class private this implements a "container" for defined access to that class. clearing up space in the global are for only those items necessary.
